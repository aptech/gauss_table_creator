# pubtableSet / ptSetup / ptSetupAt

## Purpose
> Detect installed optional GAUSS packages and write `pubtable.dec` to the correct
> directory. Run once after installing pubtable (or after installing/removing an
> optional add-on package) so that the corresponding adapters are activated.

## Format
> ret = pubtableSet()
> ret = ptSetup()
> ret = ptSetupAt(srcDir)

`pubtableSet` and `ptSetup` are aliases. `ptSetupAt` writes to a caller-supplied
directory instead of the standard package path — use it for development installs
(git clones not managed by the GAUSS package manager).

## Input
| Parameter | Description |
|:------- |:------- |
| srcDir | (`ptSetupAt` only) String, full path to the `src/` directory of the pubtable source tree where `pubtable.dec` should be written. |

## Output
| Output | Description |
|:------- |:------- |
| ret | Scalar. `0` on success; non-zero if the `.dec` file could not be written. |

## Generated file
`pubtable.dec` is machine-specific and should not be committed to source control. The
shipped `src/pubtable.dec` contains only the version declaration; the generated version
adds `#define` sentinels for each detected library.

Example generated `pubtable.dec`:
```
/* pubtable.dec
** Auto-generated by pubtableSet(). Do not edit manually. */
declare matrix _pubtable_ver = { 0, 3, 0 };

#define PT_USE_CMLMT
#define PT_USE_MAXLIKMT
#define PT_USE_TSMT
```

## Detected libraries
| Define | Package | Adapter file |
|:------- |:------- |:------- |
| `PT_USE_CMLMT` | cmlmt | `src/pubtable_cmlmt.src` |
| `PT_USE_MAXLIKMT` | maxlikmt | `src/pubtable_maxlikmt.src` |
| `PT_USE_OPTMT` | optmt | `src/pubtable_optmt.src` |
| `PT_USE_TSMT` | tsmt | `src/pubtable_tsmt.src` |

The qardl adapter is activated automatically by the `QARDL_SDF_INCLUDED` guard written
by `qardl.sdf` — no `pubtableSet` entry is needed.

## Usage after setup
For installed packages (`library pubtable;` workflow), no explicit `#include pubtable.dec` is
needed in user code at all: each optional adapter file (`pubtable_cmlmt.src`, `pubtable_maxlikmt.src`,
`pubtable_optmt.src`, `pubtable_tsmt.src`) includes `pubtable.dec` itself, right after
`#include pubtable.sdf`. Just load the optional library together with pubtable in a single
statement, e.g. `library cmlmt, pubtable;` — never as two separate `library` statements, since each
`library` statement unloads every previously loaded library not named in it.

For dev-path installs (direct `#include` instead of the package manager), include the core split
files plus any adapter files you need, in this order:
```gauss
#include C:\path\to\gauss_table_creator\src\pubtable.sdf
#include C:\path\to\gauss_table_creator\src\pubtable.src
#include C:\path\to\gauss_table_creator\src\pubtable_model.src
#include C:\path\to\gauss_table_creator\src\pubtable_render.src
#include C:\path\to\gauss_table_creator\src\pubtable_export.src
```

## Example
```gauss
new;
library pubtable;

/* Run once after install. */
call pubtableSet();

/* Subsequent programs just use 'library pubtable;' — no further setup needed. */
```

For a dev-path (git-clone) install, writing to the source directory:
```gauss
new;
#include C:\path\to\gauss_table_creator\src\pubtable.sdf
#include C:\path\to\gauss_table_creator\src\pubtable.src

call ptSetupAt("C:\\path\\to\\gauss_table_creator\\src\\");
```
